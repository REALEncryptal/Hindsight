Hindsight

The top-level module. Returned by require(path.to.Hindsight).

Hindsight exposes one factory (createWorld) plus a Defaults table for introspection. Every other interaction — casting projectiles, capturing rollback poses, querying snapshots — goes through the World and Rollback objects this factory builds. 

local Hindsight = require(ReplicatedStorage.Hindsight)

local world = Hindsight.createWorld({
    actorContainer    = ServerScriptService,
    definitionsModule = ReplicatedStorage.Shared.Definitions,
})

Types​

Extra​

</>
type Extra = {[string]: any}

An open table travelling with every projectile. Set on a per-cast basis via ProjectileModifier.extra, surfaced on every callback's ctx.extra, and passed to the Filter.

Caster​

</>
type Caster = Player | Model

Whatever the game considers the shooter. Players for player weapons; Models for NPCs, turrets, or vehicles. The reference must resolve identically on every peer that runs a matching cast.

Filter​

</>
type Filter = (
caster: Caster,
victim: Player?,
character: Model,
extra: Extra
) → boolean

Optional per-character hit-skip predicate. Returns true to skip the candidate; false/nil to accept. Lives on a ProjectileDefinition or is passed directly to Rollback:queryRay.

When used by the projectile simulator, runs inside the simulation actor in parallel context — read-only and thread-safe: no instance writes, no Humanoid state inspection beyond what's already captured.

When used by a manual Rollback:queryRay call, runs on the main thread.

Rig​

</>
interface Rig {
parts: {string}--

Names of the BaseParts that form this rig's hitboxes, indexed in parallel with sizes.

sizes: {Vector3}--

Half-extents of each part's OBB, in stud-space. sizes[i] is paired with parts[i].

}

Hitbox layout for one rig kind. See Defaults.rigs for the built-in R15 and R6 entries.

RollbackConfig​

</>
interface RollbackConfig {
lifetime: number--

Seconds of history retained. Older snapshots are dropped on capture.

voxelSize: number--

Broadphase voxel cell size in studs.

gridCenter: Vector3--

World-space center of the voxel space.

gridSize: Vector3--

World-space size of the voxel space. Characters outside this AABB are not findable via queryRay.

hitboxSize: Vector3--

Half-extents used to fatten each character's footprint in the voxel grid.

rigs: {[string]: Rig}--

Hitbox layouts, keyed by rig name (e.g. "R15", "R6").

}

PenetrationConfig​

</>
interface PenetrationConfig {
surfaceHardness: {[Enum.Material]: number}--

Per-material hardness. Unlisted materials use defaultHardness.

defaultHardness: number--

Fallback hardness for materials not in surfaceHardness.

ricochetHardness: number--

Minimum hardness for a surface to be eligible to ricochet a projectile.

mediumGapThreshold: number--

Gap size in studs below which adjacent parts are merged into one compound medium.

maxCompoundMediumParts: number--

Upper bound on parts joined into one compound medium.

}

WorldConfig​

</>
interface WorldConfig {
actorContainer: Instance--

Container into which the simulation Actors are parented.

definitionsModule: ModuleScript--

ModuleScript returning { [string]: ProjectileDefinition }.

visualsContainer: Instance?--

Where bullet visuals get parented on the client. nil on the server.

excludeContainers: {Instance}?--

Descendants are skipped as penetration continuation candidates (e.g. visuals folder, characters folder).

threads: number?--

Number of simulation actors. Defaults to 16.

serverFrameRate: number?--

Frame budget for the actor pool. Defaults to 1/60.

frameTimeBudget: number?--

Fraction of serverFrameRate an actor will spend stepping projectiles per frame. Defaults to 0.5.

interpolation: number?--

Client-side replication interpolation delay. Defaults to 0.048.

rollback: RollbackConfig?--

Overrides Defaults.rollback.

penetration: PenetrationConfig?--

Overrides Defaults.penetration.

defaultRaycastFilter: RaycastParams?--

Fallback raycast filter used when a definition doesn't supply its own.

}

The top-level configuration passed to createWorld. Every wiring decision lives here — the library never looks anything up on its own.

ProjectileDefinition​

</>
interface ProjectileDefinition {
velocity: number--

Initial speed in studs/second.

gravity: Vector3--

World-space gravity vector applied every step.

lifetime: number--

Seconds before the projectile self-destructs.

power: number--

Penetration budget. Spent against material hardness when crossing surfaces.

angle: number--

Maximum impact angle (degrees) that allows a ricochet. 360 always bounces; 0 never does.

loss: number--

Speed lost per ricochet.

collaterals: boolean?--

If true, continues through players instead of stopping on the first intersection.

raycastFilter: RaycastParams?--

Optional RaycastParams overriding World.defaultRaycastFilter.

filter: Filter?--

Per-character skip predicate. Parallel-safe.

onImpact: ((ctx: ImpactCtx) → ())?--

World-geometry hit callback.

onIntersection: ((ctx: IntersectionCtx) → ())?--

Captured-character intersection callback.

onDestroyed: ((ctx: DestroyedCtx) → ())?--

Expiration callback.

}

A single ammo type. Lives in the WorldConfig.definitionsModule. Modifier values override the numeric/data fields per cast; callbacks and filter are definition-only because functions cannot safely cross actor VM boundaries.

ProjectileModifier​

</>
interface ProjectileModifier {
velocity: number?
gravity: Vector3?
lifetime: number?
power: number?
angle: number?
loss: number?
collaterals: boolean?
raycastFilter: RaycastParams?
extra: Extra?
}

Per-cast overrides for numeric / data fields only. Functions (callbacks, filter) are not present — they live in the definitions module so each actor can require its own copy.

</>
interface CastOptions {

Whoever fired the projectile.

Key in the definitions module (e.g. "Bullet").

World-space spawn point.

Direction of travel. Normalized internally if not unit.

Server time at which the projectile started. Used for rollback queries.

Optional visual template. Cloned into World.visualsContainer on the client.

}

The single argument to World:cast.

CharacterPose​

</>
interface CharacterPose {
rig: string--

Rig key (e.g. "R15", "R6") — must match one in RollbackConfig.rigs.

rootPosition: Vector3--

World-space position used to bucket the character into the voxel broadphase.

parts: {CFrame}--

Per-part CFrames, indexed in parallel with the rig's parts list.

player: Player?--

Owning player, if any. Surfaces on RollbackHit.player and the Filter's victim argument.

}

One character's frame data. The caller builds this every tick and pushes it via Rollback:capture. Mismatched part counts are silently dropped at push time.

CharacterPoses​

</>
type CharacterPoses = {[Model]: CharacterPose}

The shape consumed by Rollback:capture.

</>
interface RollbackHit {

Owning player of the character that was hit.

character: Model--

The hit character Model.

part: string--

Name of the part that was intersected (e.g. "Head").

position: Vector3--

World-space hit point.

distance: number--

Distance from the ray origin to the hit point.

}

The shape returned by Rollback:queryRay.

Properties​

Defaults​

</>
Hindsight.Defaults: Defaults

Built-in defaults used when the caller omits a config block on createWorld. Useful for debug tooling that wants to introspect the values the world is actually running with, and as a base to clone-and-edit when you only want to override a few fields.

Functions​

createWorld​

</>
Hindsight.createWorld(config: WorldConfig) → World

Builds a new World. Spins up config.threads simulation actors (default 16) under config.actorContainer, requires the definitions module in every actor VM, builds the main-thread snapshot store, and returns the handle.

actorContainer and definitionsModule are required; every other field falls back to the values on Hindsight.Defaults. 

local world = Hindsight.createWorld({
    actorContainer    = ServerScriptService,
    definitionsModule = ReplicatedStorage.Shared.Definitions,
    visualsContainer  = workspace.Bullets,
})
